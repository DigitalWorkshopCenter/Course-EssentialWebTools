Assignment #3

Overview

This assignment will be unique, because in order to complete it, you will have to work with your peers in this class. :) In short, you will:

  • clone a Github repository

  • create a branch

  • add your code

  • retrieve/pull the code from the branches of your peers

  • resolve merge conflicts

  • create a pull request to merge the team's code with the master branch

The merge process is generally handled at an administrator level. In this case, only the owner of the repository will be able to allow changes to the master branch. But you will be able to clone the project, make a branch, then submit a pull request. When a pull request is submitted, the administrator of a repository will be able to view it, view your comments, your commits, and all code that was changed to decide whether they will merge the request and close it, and reject the merge and provide feedback as to why it was rejected.

The tricky part of this task will be to merge your code with the code of your peers. You will likely have to deal with merge conflicts and this will help you a lot with these essential skills.

Merge Conflict Practice

Since you are not the administrator of the Github Repository that this project was cloned from, you will not have to sort out merge conflicts when merging branches. When you do a git pull, frequently you will have to deal with merge conflicts with code you've written and code that was saved on the master branch. But in this part of the activity, you are going to make another modification in index.html, then pull the code from another branch into your branch. This should give you experience working with merge conflicts. Let's get started. :)

  1. Make a couple more changes to index.html. Ensure that the modal for your project contains a link to your project for others to access it. Also, modify the description of your project in your modal (even if it is just a single character that you change). If there is anything else you'd like to modify in the page, you may.

  2. Commit your changes and push your branch's updates to Github.

  3. Now, you are going to retrieve code from the branch of another student.

    • Go to https://github.com/nathanbirch/html-template2/branches and make note of what branches are there. You may pull from the studentChanges branch, or the branch of someone in your class.

    • To do this, you can use the command git pull origin studentChanges, or you can use the tool in VS Code. Click the 3 dots > then pull/push > push from> select --the branch you want to use--

    • Inspect any errors that take place, and try to fix them.

    • You will not create a pull request with this updated code from you and another student. This is just for you to get experience working with merge conflicts.
